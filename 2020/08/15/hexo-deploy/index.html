Deploying the Hexo website to Github

Posted on 2020-08-15 Edited on 2025-04-02

Creating the user repo on Github

Assuming we already have a Github account, we need to first log in to our personal page, then create a personal repository. Note that the name of this repository needs to strictly follow the format githubID.github.io, or else our hexo server will fail to deploy to Github.

Deployment

Now, we can actually connect the hexo server and the Github repo. This step will upload the files generated by hexo into the GitHub repo, so we can access the website publicly through the GitHub server. To do this, we first add the relevant deployment information in the _config.yml page. We can edit it using sudo geany _config.yml.

Now we need to install the deployment software as well, using npm install hexo-deployer-git --save.

If we now visit http://id.github.io we will be able to see the website. However, this is a rather random address to remember, so we can give the website a personalised address. We can do this by buying a domain name such as andycai.top from vendors such as godaddy or alicloud. A more common suffix like .com may be more expensive. After buying the domain name, we can set up an CNAME (canonical Name record) for the domain, essentially making it an alias of our default id.github.io domain.

Now, we can access our website from the new domain name andycai.top.
